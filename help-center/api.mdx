# Help Center API

All help center endpoints are under `/api/v1/kb/` and require the `helpCenter` feature flag to be enabled. Use the same `qb_` API keys you use for the rest of the Quackback API.

<Warning>
The help center is an experimental feature. Enable the `helpCenter` flag in **Admin > Settings > Experimental** before using any help center features.
</Warning>

## Authentication

Include your API key in the `Authorization` header on every request:

```
Authorization: Bearer qb_your_api_key
```

[Create an API key](/admin/api-keys) in **Admin > Settings > API Keys**. Team role is required for read operations; admin role is required for writes.

## Categories

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v1/kb/categories` | List all categories |
| `POST` | `/api/v1/kb/categories` | Create a category |
| `GET` | `/api/v1/kb/categories/:id` | Get a category |
| `PATCH` | `/api/v1/kb/categories/:id` | Update a category |
| `DELETE` | `/api/v1/kb/categories/:id` | Delete a category |

### Create a category

**Request body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Display name |
| `slug` | string | Yes | URL-safe identifier, e.g. `getting-started` |
| `description` | string | No | Short description shown on the help center homepage |
| `icon` | string | No | Icon name |
| `isPublic` | boolean | No | Default: `true` |
| `position` | number | No | Sort order |

**Response fields:** `id`, `slug`, `name`, `description`, `icon`, `isPublic`, `position`, `articleCount`, `createdAt`, `updatedAt`

<CodeGroup>
  <CodeBlock title="curl">
  ```bash
  curl https://feedback.example.com/api/v1/kb/categories \
    -H "Authorization: Bearer qb_your_api_key" \
    -H "Content-Type: application/json" \
    -d '{
      "name": "Getting Started",
      "slug": "getting-started",
      "description": "Everything you need to get up and running",
      "isPublic": true,
      "position": 1
    }'
  ```
  </CodeBlock>

  <CodeBlock title="JavaScript">
  ```javascript
  const response = await fetch('https://feedback.example.com/api/v1/kb/categories', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer qb_your_api_key',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      name: 'Getting Started',
      slug: 'getting-started',
      description: 'Everything you need to get up and running',
      isPublic: true,
      position: 1,
    }),
  });

  const category = await response.json();
  ```
  </CodeBlock>

  <CodeBlock title="Python">
  ```python
  import requests

  response = requests.post(
      "https://feedback.example.com/api/v1/kb/categories",
      headers={"Authorization": "Bearer qb_your_api_key"},
      json={
          "name": "Getting Started",
          "slug": "getting-started",
          "description": "Everything you need to get up and running",
          "isPublic": True,
          "position": 1,
      },
  )

  category = response.json()
  ```
  </CodeBlock>
</CodeGroup>

## Articles

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/v1/kb/articles` | List articles (paginated) |
| `POST` | `/api/v1/kb/articles` | Create an article |
| `GET` | `/api/v1/kb/articles/:id` | Get an article |
| `PATCH` | `/api/v1/kb/articles/:id` | Update an article |
| `DELETE` | `/api/v1/kb/articles/:id` | Delete an article |
| `POST` | `/api/v1/kb/articles/:id/feedback` | Record helpful / not helpful |

### List articles

**Query parameters:**

| Parameter | Type | Description |
|-----------|------|-------------|
| `categoryId` | string | Filter by category |
| `status` | `draft` \| `published` \| `all` | Filter by status. Defaults to `published` |
| `search` | string | Full-text search query |
| `cursor` | string | Pagination cursor from previous response |
| `limit` | number | Results per page. Range: 1-100, default: 20 |

### Create an article

**Request body:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `categoryId` | string | Yes | ID of the parent category |
| `title` | string | Yes | Article title |
| `content` | string | No | Article body (HTML or Markdown) |
| `slug` | string | No | URL-safe identifier. Auto-generated from title if omitted |
| `description` | string | No | Short summary shown in search results |

<CodeGroup>

### Update an article

**Request body** — all fields optional:

| Field | Type | Description |
|-------|------|-------------|
| `title` | string | Article title |
| `content` | string | Article body |
| `slug` | string | URL identifier |
| `description` | string | Short summary |
| `categoryId` | string | Move to a different category |
| `publishedAt` | string \| null | ISO 8601 date to publish; `null` to unpublish |

Set `publishedAt` to an ISO 8601 date string to publish an article, or `null` to revert it to draft.

<CodeGroup>
  <CodeBlock title="curl">
  ```bash
  # Publish an article
  curl -X PATCH https://feedback.example.com/api/v1/kb/articles/kbart_01h4... \
    -H "Authorization: Bearer qb_your_api_key" \
    -H "Content-Type: application/json" \
    -d '{"publishedAt": "2026-04-10T12:00:00Z"}'
  ```
  </CodeBlock>

  <CodeBlock title="JavaScript">
  ```javascript
  // Publish an article
  const response = await fetch('https://feedback.example.com/api/v1/kb/articles/kbart_01h4...', {
    method: 'PATCH',
    headers: {
      'Authorization': 'Bearer qb_your_api_key',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ publishedAt: new Date().toISOString() }),
  });
  ```
  </CodeBlock>

  <CodeBlock title="Python">
  ```python
  from datetime import datetime, timezone
  import requests

  # Publish an article
  response = requests.patch(
      "https://feedback.example.com/api/v1/kb/articles/kbart_01h4...",
      headers={"Authorization": "Bearer qb_your_api_key"},
      json={"publishedAt": datetime.now(timezone.utc).isoformat()},
  )
  ```
  </CodeBlock>
</CodeGroup>

### Record article feedback

Let users signal whether an article helped them. POST to the feedback endpoint with a boolean:

```json
{ "helpful": true }
```

This data appears in the admin article list so you can identify articles that need improvement.

### Search articles

Use the `search` query parameter on the list endpoint to run a full search query:

<CodeGroup>
  <CodeBlock title="curl">
  ```bash
  curl "https://feedback.example.com/api/v1/kb/articles?search=reset+password&limit=5" \
    -H "Authorization: Bearer qb_your_api_key"
  ```
  </CodeBlock>

  <CodeBlock title="JavaScript">
  ```javascript
  const params = new URLSearchParams({ search: 'reset password', limit: '5' });
  const response = await fetch(
    `https://feedback.example.com/api/v1/kb/articles?${params}`,
    { headers: { 'Authorization': 'Bearer qb_your_api_key' } },
  );

  const { articles } = await response.json();
  ```
  </CodeBlock>

  <CodeBlock title="Python">
  ```python
  import requests

  response = requests.get(
      "https://feedback.example.com/api/v1/kb/articles",
      headers={"Authorization": "Bearer qb_your_api_key"},
      params={"search": "reset password", "limit": 5},
  )

  articles = response.json()["articles"]
  ```
  </CodeBlock>
</CodeGroup>

## Widget search

A public endpoint powers search inside the embedded widget. No authentication required.

```
GET /api/widget/kb-search?q=your+query&limit=5
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `q` | string | Search query |
| `limit` | number | Max results to return |

**Response** — array of matching articles with `title`, `description`, `slug`, and `category`.

<Note>
This endpoint only returns published articles with `isPublic: true`. Draft and private articles are never exposed through the widget.
</Note>
